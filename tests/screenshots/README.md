# Screenshot Regression Testing Suite

## Overview

This directory contains a comprehensive screenshot capture system for regression testing of the Agent Feed application running on localhost:5173.

## Quick Start

1. **Start the development server:**
   ```bash
   cd frontend && npm run dev
   ```

2. **Capture baseline screenshots:**
   ```bash
   node tests/screenshots/capture-runner.js
   ```

3. **Validate captured screenshots:**
   ```bash
   node tests/screenshots/validate-capture.js
   ```

## Files

- `regression-capture.spec.ts` - Main Playwright test suite for screenshot capture
- `screenshot-config.ts` - Playwright configuration with multiple viewport support
- `capture-runner.js` - Node.js script to orchestrate the entire capture process
- `validate-capture.js` - Validation script to check captured screenshots
- `regression-analysis-report.md` - Detailed analysis report

## Usage in CI/CD

The screenshot suite can be integrated into CI/CD pipelines:

```yaml
- name: Capture Screenshots
  run: |
    npm run dev &
    sleep 10
    node tests/screenshots/capture-runner.js

- name: Validate Screenshots
  run: node tests/screenshots/validate-capture.js
```

## Regression Testing Workflow

1. **Baseline Capture**: Run initial screenshot capture to establish baseline
2. **Code Changes**: Make application changes
3. **New Capture**: Run screenshot capture again
4. **Visual Diff**: Compare new screenshots with baseline
5. **Review**: Approve intentional changes, fix regressions

## Configuration

### Playwright Settings
- Timeout: 60 seconds per test
- Wait strategy: Network idle + image loading
- Headless mode: Required for Codespace environment
- Screenshot format: PNG with standard compression

### Environment Requirements
- Node.js 18+
- Development server running on localhost:5173
- Playwright browsers installed
