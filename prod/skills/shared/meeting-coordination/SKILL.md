# Meeting Coordination Skill

## Purpose

Provides systematic frameworks for all aspects of meeting management: preparation, agenda setting, note-taking, action item capture, and post-meeting follow-through. Transforms meetings from time sinks into productive decision-making and coordination vehicles.

## When to Use This Skill

- Preparing for upcoming meetings
- Creating and distributing meeting agendas
- Capturing and distributing meeting notes
- Extracting and tracking action items
- Following up on meeting commitments
- Evaluating meeting effectiveness

## Core Meeting Frameworks

### 1. Meeting Classification Framework

**Meeting Types and Purposes**:

```
ONE-ON-ONE (1:1):
  Purpose: Personal connection, coaching, feedback, alignment
  Duration: 30-45 minutes
  Frequency: Weekly or bi-weekly
  Participants: Manager + Direct report
  Key Outputs: Action items, feedback delivered, alignment confirmed

TEAM STANDUP:
  Purpose: Daily synchronization, blocker identification
  Duration: 15 minutes maximum
  Frequency: Daily
  Participants: Core team
  Key Outputs: Progress updates, blockers flagged, quick decisions

PROJECT STATUS:
  Purpose: Track progress, identify risks, make decisions
  Duration: 45-60 minutes
  Frequency: Weekly or bi-weekly
  Participants: Project team + stakeholders
  Key Outputs: Status dashboard, risk register, decision log

STRATEGIC PLANNING:
  Purpose: Long-term planning, goal setting, resource allocation
  Duration: 2-4 hours
  Frequency: Quarterly
  Participants: Leadership team
  Key Outputs: Strategic plan, OKRs, resource commitments

DECISION-MAKING:
  Purpose: Make specific high-stakes decisions
  Duration: 60-90 minutes
  Frequency: As needed
  Participants: Decision makers + subject matter experts
  Key Outputs: Formal decision, rationale documented

BRAINSTORMING / IDEATION:
  Purpose: Generate creative solutions, explore possibilities
  Duration: 60-90 minutes
  Frequency: As needed
  Participants: Diverse perspectives, relevant stakeholders
  Key Outputs: Idea catalog, prioritized concepts, next steps

RETROSPECTIVE / POST-MORTEM:
  Purpose: Learn from completed work, identify improvements
  Duration: 60-90 minutes
  Frequency: End of project/sprint
  Participants: Team members, key stakeholders
  Key Outputs: Lessons learned, improvement actions, celebration
```

### 2. Meeting Preparation Framework

**Pre-Meeting Checklist (24 hours before)**:
```
✓ AGENDA distributed to all participants
✓ PURPOSE and desired outcomes clearly stated
✓ PRE-READING materials shared (if required)
✓ ATTENDEES confirmed and appropriate for meeting purpose
✓ DURATION respect for everyone's calendar time
✓ LOGISTICS confirmed (location/link, tech setup)
✓ TIME ZONE adjustments noted for distributed teams
✓ PREVIOUS ACTION ITEMS reviewed and status prepared
```

**Agenda Template**:
```
MEETING: [Meeting Title]
DATE: [Date and time with timezone]
DURATION: [Expected duration]
LOCATION/LINK: [Physical location or video link]
FACILITATOR: [Person running the meeting]
NOTE-TAKER: [Person responsible for notes]

PURPOSE:
[One clear sentence on why we're meeting]

DESIRED OUTCOMES:
- [Specific outcome 1]
- [Specific outcome 2]
- [Specific outcome 3]

AGENDA:
1. [Topic 1] - [Time allocation] minutes
   - [Sub-point if needed]
   Owner: [Who leads this discussion]

2. [Topic 2] - [Time allocation] minutes
   Owner: [Who leads this discussion]

3. [Topic 3] - [Time allocation] minutes
   Owner: [Who leads this discussion]

4. Action Items Review - 5 minutes
   - Review previous action items
   - Capture new action items

5. Next Steps & Closing - 5 minutes
   - Summarize decisions and actions
   - Confirm next meeting if recurring

PRE-READING (if applicable):
- [Document 1 - link]
- [Document 2 - link]

ATTENDEES:
- [Name 1] - [Role]
- [Name 2] - [Role]
```

**Meeting Invitation Best Practices**:
```
SUBJECT LINE: [Meeting Type]: [Clear Purpose] - [Date]
  Good: "Decision: Q4 Budget Allocation - Oct 20"
  Bad: "Meeting"

DESCRIPTION:
  - Purpose in first sentence
  - Desired outcomes bulleted
  - Preparation required clearly stated
  - Agenda attached or inline
  - Pre-reading linked

ATTENDEES:
  - Required: Only those who must attend for meeting to succeed
  - Optional: Those who might benefit but aren't critical
  - FYI: None - don't waste calendar space

TIMING:
  - Avoid: Early morning, late evening, lunch hours if possible
  - Prefer: Mid-morning or mid-afternoon
  - Respect: Time zones for distributed teams
```

### 3. Meeting Execution Framework

**Opening (First 5 minutes)**:
```
1. WAIT for all attendees (2 minutes grace period max)
2. REVIEW agenda and desired outcomes
3. CONFIRM note-taker and timekeeper
4. SET ground rules if needed (for larger or sensitive meetings)
5. START recording if remote and agreed upon
```

**During Meeting Best Practices**:
```
FACILITATOR RESPONSIBILITIES:
✓ Keep discussion on track and on time
✓ Ensure all voices heard (draw out quiet participants)
✓ Manage dominant voices (politely redirect)
✓ Summarize key points periodically
✓ Capture decisions explicitly
✓ Assign action items with owners and deadlines
✓ Watch time and adjust agenda if needed

NOTE-TAKER RESPONSIBILITIES:
✓ Document key discussion points (not verbatim)
✓ Record all decisions made
✓ Capture action items with 3 W's: Who, What, When
✓ Note open questions or parking lot items
✓ Track attendance
✓ Timestamp important moments for recording reference
```

**Parking Lot Technique**:
```
PURPOSE: Capture important but off-topic items without derailing meeting

WHEN TO USE:
- Discussion going off-topic but issue is valid
- Good idea that needs more time than available
- Question requires research before answering
- Scope creep beyond meeting purpose

HOW TO USE:
1. ACKNOWLEDGE: "That's an important point"
2. CAPTURE: Add to parking lot list with brief description
3. COMMIT: "Let's follow up on this separately"
4. REDIRECT: "For now, let's get back to [topic]"

FOLLOW-UP:
- Review parking lot at end of meeting
- Assign ownership for follow-up
- Schedule separate discussions if needed
- Don't let parking lot items disappear
```

### 4. Meeting Notes Framework

**Meeting Notes Template**:
```
# [Meeting Title]

**Date**: [Date and time]
**Attendees**: [Names] (mark any absences)
**Facilitator**: [Name]
**Note-Taker**: [Name]

## Purpose
[Why we met - one sentence]

## Key Discussion Points
- [Major point 1 with relevant context]
- [Major point 2 with relevant context]
- [Major point 3 with relevant context]

## Decisions Made
1. **[Decision topic]**
   - Decision: [What was decided]
   - Rationale: [Why this decision]
   - Owner: [Who executes]
   - Timeline: [When]

2. **[Decision topic 2]**
   [Same structure]

## Action Items
| # | Action | Owner | Due Date | Status |
|---|--------|-------|----------|--------|
| 1 | [Specific action] | [Name] | [Date] | Pending |
| 2 | [Specific action] | [Name] | [Date] | Pending |

## Open Questions
- [Question 1] - Owner: [Who will research]
- [Question 2] - Owner: [Who will research]

## Parking Lot Items
- [Item 1] - Follow-up: [How/when addressed]
- [Item 2] - Follow-up: [How/when addressed]

## Next Steps
- [Immediate next step 1]
- [Immediate next step 2]

## Next Meeting
**Date**: [If recurring]
**Agenda Preview**: [Key topics for next meeting]

---
**Notes Distribution**: [Date distributed]
**Recording Link**: [If applicable]
```

**Notes Distribution Protocol**:
```
TIMING: Within 24 hours of meeting (ideally within 4 hours)

RECIPIENTS:
- All attendees (required and optional)
- Key stakeholders who couldn't attend
- Action item owners (if not attendees)

FORMAT:
- Markdown or well-formatted document
- PDF for archival purposes
- Accessible link (not attachment if possible)

STORAGE:
- Centralized location (wiki, shared drive, project management tool)
- Consistent naming: "YYYY-MM-DD - [Meeting Title] - Notes"
- Searchable and version-controlled
```

### 5. Action Item Management Framework

**Well-Formed Action Item Structure**:
```
Every action item MUST have:

WHAT: Specific, actionable description
  ✓ Good: "Create draft Q4 budget proposal with 3 scenarios"
  ✗ Bad: "Think about budget"

WHO: Single person accountable (not a team)
  ✓ Good: "Sarah Johnson"
  ✗ Bad: "Engineering team"

WHEN: Specific deadline with date
  ✓ Good: "October 25, 2025 EOD"
  ✗ Bad: "Next week"

Optional but valuable:
WHY: Business context (for prioritization)
DEPENDENCIES: Blockers or prerequisites
SUPPORT: Who can help if needed
```

**Action Item Tracking**:
```
STATUS TYPES:
- Not Started: Deadline not yet approached
- In Progress: Owner actively working
- Blocked: Dependency or issue preventing progress
- Completed: Done and verified
- Deferred: Postponed with new timeline
- Cancelled: No longer relevant

TRACKING CADENCE:
- Review in next meeting of same type
- For critical items: Mid-point check-in
- For overdue items: Immediate follow-up
```

**Action Item Handoff Protocol**:
```
MEETING-NEXT-STEPS-AGENT responsibilities:
1. EXTRACT all action items from notes
2. VERIFY each has WHO, WHAT, WHEN
3. CREATE tracking entries
4. NOTIFY owners of their assignments
5. HAND OFF to follow-ups-agent for:
   - Delegation tracking
   - Progress monitoring
   - Escalation if needed
```

### 6. Meeting Effectiveness Framework

**Post-Meeting Evaluation**:
```
MEETING SCORE (1-10 scale):

PURPOSE CLARITY (Weight: 20%):
- Was the purpose clear upfront?
- Did we achieve stated outcomes?

TIME EFFICIENCY (Weight: 25%):
- Did we start/end on time?
- Was time allocated well across topics?
- Could this have been shorter or an email?

ENGAGEMENT (Weight: 20%):
- Were all attendees engaged and participating?
- Were the right people present?

DECISIONS & ACTIONS (Weight: 25%):
- Were decisions made that needed to be made?
- Are action items clear and assigned?

FOLLOW-THROUGH (Weight: 10%):
- Were previous action items reviewed?
- Is follow-up plan clear?

OVERALL SCORE: [Weighted average]

IMPROVEMENT ACTIONS:
- What would make next meeting better?
- What should we stop/start/continue?
```

**Meeting ROI Calculation**:
```
MEETING COST:
  Number of attendees: X
  Average hourly rate: $Y
  Meeting duration: Z hours
  Total cost: X × $Y × Z = $TOTAL

MEETING VALUE:
  - Decisions made: [List]
  - Problems solved: [List]
  - Alignment achieved: [List]
  - Action items generated: [List]
  - Estimated value delivered: $VALUE

ROI: (VALUE - COST) / COST × 100%

RULE: If ROI consistently negative → Meeting should be:
- Eliminated entirely
- Converted to asynchronous communication
- Radically restructured
- Reduced in frequency or duration
```

### 7. Meeting Alternatives Framework

**When NOT to Meet**:
```
USE EMAIL/DOCUMENT Instead when:
✓ Information is one-way (announcement, update)
✓ No discussion or debate needed
✓ Attendees need time to digest before responding
✓ Asynchronous responses acceptable

USE INSTANT MESSAGE Instead when:
✓ Quick question with quick answer
✓ Yes/no decision from single person
✓ Scheduling or logistics coordination
✓ Brief status update

USE COLLABORATIVE DOCUMENT Instead when:
✓ Brainstorming ideas (can comment asynchronously)
✓ Gathering input from many people
✓ Building consensus on written content
✓ Long-term decision documentation

HAVE THE MEETING when:
✓ Complex discussion requiring back-and-forth
✓ High-stakes decision needing consensus
✓ Conflict resolution or difficult conversation
✓ Relationship building or team bonding
✓ Rapid decision-making required
```

## Best Practices

### For Meeting Preparation:
1. **Question the Meeting**: Does this HAVE to be a meeting?
2. **Invite Intentionally**: Fewer attendees = more productive
3. **Prepare Thoroughly**: Agenda 24 hours in advance minimum
4. **Set Clear Outcomes**: Know what success looks like
5. **Share Context**: Pre-reading prevents wasted meeting time

### For Meeting Execution:
1. **Start On Time**: Respect everyone's calendar
2. **Assign Roles**: Facilitator and note-taker explicitly named
3. **Stay On Track**: Parking lot for tangents
4. **End Early**: If outcomes achieved, don't fill time
5. **Capture Actions**: WHO, WHAT, WHEN for every action

### For Meeting Follow-up:
1. **Distribute Quickly**: Notes within 24 hours
2. **Track Religiously**: Action items don't disappear
3. **Close Loops**: Report back on completed actions
4. **Learn Continuously**: Evaluate and improve
5. **Say No**: Decline meetings where you don't add value
