# Agent Feed - Complete System Architecture & Deployment Guide\n\n## Overview\n\nThis document provides comprehensive instructions for deploying the Agent Feed system using the complete architecture setup. The system is designed as a scalable microservices architecture with comprehensive monitoring, security, and development tooling.\n\n## Architecture Summary\n\n### Services Architecture\n- **API Gateway** (Port 3000): Request routing, authentication, rate limiting\n- **Agent Management** (Port 3001): Agent lifecycle and coordination\n- **Feed Processing** (Port 3002): Content aggregation and processing\n- **User Management** (Port 3003): Authentication and user profiles\n- **WebSocket Service** (Port 3004): Real-time communication\n- **Monitoring** (Port 3005): Metrics and health monitoring\n\n### Infrastructure Components\n- **PostgreSQL 15**: Primary database\n- **Redis 7**: Caching and session storage\n- **Nginx**: Reverse proxy and load balancer\n- **Prometheus**: Metrics collection\n- **Grafana**: Monitoring dashboards\n\n### Development & Operations\n- **TypeScript**: Type-safe development\n- **Jest**: Comprehensive testing framework\n- **ESLint**: Code quality and security\n- **Docker**: Containerization\n- **Kubernetes**: Production orchestration\n- **GitHub Actions**: CI/CD pipeline\n\n## Quick Start (Development)\n\n### Prerequisites\n- Node.js 18+\n- Docker & Docker Compose\n- Git\n\n### Setup Development Environment\n\n```bash\n# 1. Clone and setup\ngit clone <repository>\ncd agent-feed\n\n# 2. Run automated setup\n./scripts/setup-dev.sh\n\n# 3. Start development services\nnpm run dev\n```\n\n### Manual Setup (if needed)\n\n```bash\n# 1. Install dependencies\nnpm ci\n\n# 2. Setup environment\ncp .env.example .env\n# Edit .env with your configuration\n\n# 3. Start infrastructure\ndocker-compose -f infrastructure/docker/docker-compose.yml up -d postgres redis\n\n# 4. Run database migrations\nnpm run migrate:dev\n\n# 5. Start services\nnpm run build\nnpm start\n```\n\n## Production Deployment\n\n### Option 1: Docker Compose (Simple)\n\n```bash\n# 1. Set production environment variables\nexport NODE_ENV=production\nexport POSTGRES_PASSWORD=secure_password\nexport REDIS_PASSWORD=secure_redis_password\nexport JWT_SECRET=secure_jwt_secret_32_chars_min\n\n# 2. Deploy with Docker Compose\ndocker-compose -f infrastructure/docker/docker-compose.yml up -d\n\n# 3. Verify deployment\ncurl http://localhost/health\n```\n\n### Option 2: Kubernetes (Recommended)\n\n```bash\n# 1. Create namespace and secrets\nkubectl apply -f infrastructure/kubernetes/namespace.yaml\n\n# 2. Update secrets with actual values\n# Edit infrastructure/kubernetes/secrets.yaml\nkubectl apply -f infrastructure/kubernetes/secrets.yaml\n\n# 3. Deploy configuration\nkubectl apply -f infrastructure/kubernetes/configmap.yaml\n\n# 4. Deploy database services\nkubectl apply -f infrastructure/kubernetes/database.yaml\n\n# 5. Wait for databases to be ready\nkubectl wait --for=condition=ready pod -l app=postgres -n agent-feed --timeout=300s\nkubectl wait --for=condition=ready pod -l app=redis -n agent-feed --timeout=300s\n\n# 6. Deploy application services\nkubectl apply -f infrastructure/kubernetes/services.yaml\n\n# 7. Deploy monitoring\nkubectl apply -f infrastructure/kubernetes/monitoring.yaml\n\n# 8. Deploy ingress\nkubectl apply -f infrastructure/kubernetes/ingress.yaml\n\n# 9. Verify deployment\nkubectl get pods -n agent-feed\nkubectl get services -n agent-feed\n```\n\n### Option 3: Cloud Provider Specific\n\n#### AWS EKS\n```bash\n# 1. Database Initialization\n\n```bash\n# Create database\npsql -h localhost -U postgres -c \"CREATE DATABASE agent_feed;\"\n\n# Create user\npsql -h localhost -U postgres -c \"CREATE USER agent_feed_user WITH PASSWORD 'agent_feed_pass';\"\n\n# Grant permissions\npsql -h localhost -U postgres -c \"GRANT ALL PRIVILEGES ON DATABASE agent_feed TO agent_feed_user;\"\n\n# Run migrations\nnpm run migrate:latest\n```\n\n### Backup and Restore\n\n```bash\n# Backup\npg_dump -h localhost -U agent_feed_user -d agent_feed > backup.sql\n\n# Restore\npsql -h localhost -U agent_feed_user -d agent_feed < backup.sql\n```\n\n## Security Configuration\n\n### SSL/TLS Setup\n\n```bash\n# Generate self-signed certificates (development only)\nopenssl req -x509 -newkey rsa:4096 -keyout app=api-gateway -n agent-feed\n   \n   # Check environment variables\n   docker-compose config\n   kubectl describe configmap agent-feed-config -n agent-feed\n   ```\n\n2. **Database connection issues**\n   ```bash\n   # Test database connection\n   psql -h localhost -U agent_feed_user -d agent_feed -c \"SELECT NOW();\"\n   \n   # Check database status\n   docker-compose exec postgres pg_isready\n   kubectl exec -it $(kubectl get pod -l app=postgres -n agent-feed -o jsonpath='{.items[0].metadata.name}') -n agent-feed -- pg_isready\n   ```\n\n3. **Redis connection issues**\n   ```bash\n   # Test Redis connection\n   redis-cli -h localhost -p 6379 ping\n   \n   # Check Redis status\n   docker-compose exec redis redis-cli ping\n   kubectl exec -it $(kubectl get pod -l app=redis -n agent-feed -o jsonpath='{.items[0].metadata.name}') -n agent-feed -- redis-cli ping\n   ```\n\n### Performance Optimization\n\n1. **Database Performance**\n   ```sql\n   -- Check slow
